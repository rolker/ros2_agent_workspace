# Agent Workflow Index

Machine-readable registry of all available workflows. Use this to discover what tasks are already automated.

> **Quick Reference**: For most common tasks, run `/check-status`, `/build-all`, or `/clean`

## Ops Workflows

Maintenance and operational tasks.

### Check & Monitor

| Command | Description |
|---------|-------------|
| `/check-status` | Comprehensive workspace status check (Local + GitHub) |
| `/check-local-status` | Check the local git status of the workspace |
| `/check-github-status` | Check for open Pull Requests and Issues in overlay repositories |

### Build & Compile

| Command | Description |
|---------|-------------|
| `/build` | Build ROS2 Workspaces |
| `/build-all` | Build all workspace layers in the correct order |
| `/rebuild-all` | Clean all build artifacts then build all layers |

### Maintenance

| Command | Description |
|---------|-------------|
| `/clean` | Clean build artifacts and prune removed repositories |

## Scripts Reference

For detailed information about helper scripts, see [`.agent/scripts/README.md`](.agent/scripts/README.md).

Common scripts:
- `./.agent/scripts/setup.sh <layer>` — Initialize a workspace layer
- `./.agent/scripts/status_report.sh` — Generate workspace status report
- `./.agent/scripts/env.sh` — Source the ROS2 environment
- `./.agent/scripts/configure_git_identity.sh` — Set git identity for commits

"Run the check-status workflow"

## Common Patterns

### First time in workspace?
1. Run `/check-status` to see current state
2. Check [`.agent/ROADMAP.md`](.agent/ROADMAP.md) for active tasks
3. Run `/build-all` if needed

### Before committing?
1. Run `/check-status` to verify no uncommitted changes
2. Verify your feature branch is active (`git branch`)

### Troubleshooting?
1. Run `/check-status` to diagnose
2. Review [README.md troubleshooting section](README.md#troubleshooting)
